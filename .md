## 2. Machine Learning (ML)
First introduced by **Arthur Samuel in 1959**, ML is a subset of AI concerned with the development of algorithms that allow computers to learn from data and past experience on their own.

### Applications of ML
* **Daily Tech:** Traffic alerts, Social Media, Google Translate.
* **Transportation:** Self-driving cars, Transportation computing.
* **Security & Business:** Product recommendations, Fraud detection, Dynamic programming.

---

## 3. Types of Machine Learning

### Supervised Learning
The machine is trained using **labeled input data** and corresponding outputs.
* **Classification:** Output variable is **categorical** (e.g., Yes/No, Male/Female, Red/Blue).
  * *Examples:* Spam detection, email filtering.
* **Regression:** A linear relationship between input and output variables for continuous values.
  * *Examples:* Market trends, weather prediction.
* **Applications:** Fraud detection, spam filtering, risk assessment, image segmentation.

### Unsupervised Learning
The machine is trained using **unlabeled datasets**, and it predicts output without any supervision.
* **Clustering:** Groups objects such that items with the most similarities stay in one group.
  * *Example:* Grouping customers by purchasing behavior.
* **Association:** Finds rules and dependencies between variables in large datasets.
  * *Example:* Market basket analysis (mapping item dependencies for profit).
* **Dimension Reduction:** The process of reducing the number of input variables or features in a dataset.

### Semi-Supervised Learning
Lies between supervised and unsupervised algorithms.
* **Analogy:** A student self-analyzing at home (unsupervised) while also learning under the guidance of an instructor (supervised).

### Reinforcement Learning (RL)
Works on a **feedback-based process**. An AI agent explores its surroundings through trial and error, learning from experience.
* **Mechanism:** The agent is **rewarded** for good actions and **punished** for bad actions.
* **Positive RL:** Increasing the tendency of a behavior toward a positive condition.
* **Negative RL:** Increasing the tendency of a behavior toward a negative condition.
* **Applications:** Video games, resource management, robotics.

---

## 4. Linear Regression
Establishes a linear relationship between a dependent variable ($y$) and one or more independent variables ($x$).

* **Goal:** To find the **Best Fit Line**—the line where the error between predicted values and actual values is minimized.
* **Linear Relationship:** Describes how the value of the dependent variable changes according to the independent variable.

### Types of Linear Regression
| Type | Description |
| :--- | :--- |
| **Simple Linear Regression** | Only one independent variable is present. |
| **Multiple Linear Regression** | More than one independent variable is used to find the relationship. |
| **Positive Linear Relationship** | $y$ increases as $x$ increases. |
| **Negative Linear Relationship** | $y$ decreases as $x$ increases. |

---

## 5. Naive Bayes Classifier
A supervised classification algorithm based on **Bayes’ Theorem**. It predicts the category of a data point using probability.

* **Why "Naive"?** It makes a strong assumption that all features in the dataset are **independent** of each other.
* **Bayes’ Theorem:** Calculates the posterior probability of an event based on prior knowledge.
$$P(A|B) = \frac{P(B|A) \cdot P(A)}{P(B)}$$
* **Use Case:** To predict if it's fit to play golf on a "Sunny" day with "Mild" temperature, it compares $P(Yes|Features)$ vs $P(No|Features)$ and chooses the higher probability.

# Supervised Learning: Naive Bayes Classification

## 1. What is the Naive Bayes Classifier?
The **Naive Bayes classifier** is a supervised learning algorithm used for solving classification problems. It is a probabilistic model, meaning it predicts the category of a data point by calculating and comparing probabilities.

* **Core Logic:** It assigns new data points to the most likely class by determining which class has the highest probability given the input features.
* **Applications:** Widely used in real-world tasks like spam filtering, document categorization, and sentiment analysis.

---

## 2. Why is it called "Naive"?
The algorithm is called **"Naive"** because it makes a **strong (and often unrealistic) assumption**: it assumes that all features in the dataset are **completely independent** of each other. 

> **Example:** In a fruit classification model, an apple may be identified by its color (red), shape (round), and diameter (3 inches). Naive Bayes assumes that the red color contributes to the probability of it being an apple independently of its shape or size, even if these features are actually related.

---

## 3. Bayes’ Theorem
Bayes’ Theorem is the mathematical foundation of this algorithm. It calculates the **posterior probability** of an event occurring based on prior knowledge and evidence.



### The Mathematical Formula
The theorem is expressed as:

$$P(A|B) = \frac{P(B|A) \cdot P(A)}{P(B)}$$

**Where:**
* **$P(A|B)$ (Posterior Probability):** The probability of hypothesis $A$ given the evidence $B$.
* **$P(B|A)$ (Likelihood):** The probability of evidence $B$ given that hypothesis $A$ is true.
* **$P(A)$ (Prior Probability):** The initial probability of hypothesis $A$ before seeing the evidence.
* **$P(B)$ (Marginal Likelihood):** The total probability of the evidence $B$.

---

## 4. How it Works (Step-by-Step)
In a classification task, we use the formula to find the probability of a **Class ($y$)** given a set of **Features ($x$ nodes)**:

$$P(y|x_1, x_2, ..., x_n) = \frac{P(x_1, x_2, ..., x_n | y) \cdot P(y)}{P(x_1, x_2, ..., x_n)}$$

### Practical Example: To Play Golf or Not?
If we want to predict whether it is fit to play golf on a **Sunny** day with **Mild** temperature, the classifier computes two distinct probabilities:

1.  **Probability of playing:** $P(Play = Yes \mid Sunny, Mild)$
2.  **Probability of not playing:** $P(Play = No \mid Sunny, Mild)$

**Decision Rule:**
* If $P(Yes) > P(No)$, the prediction is **Yes**.
* If $P(No) > P(Yes)$, the prediction is **No**.
